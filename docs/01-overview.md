# 01. Обзор проекта

> Проект **в разработке** (не «сбор требований»). Живой сайт: https://kissheaven9.github.io/Deutsch/ . Запуск/деплой — см. [README](../README.md).

## Цель
Веб-сайт для запоминания немецкой лексики (старт — уровень А1). Проблема пользователя: лексика в учебнике размазана по всему курсу, слова и глаголы смешаны, род путается. Решение — единая система, где слова разбиты по группам, темам и роду, и запоминаются через истории + визуализацию + звук.

Сейчас — под себя. Перспектива — открыть для других, монетизировать.

## Метод (ядро продукта)
Привязка рода к персонажу + сюжетная мнемоника + мультисенсорика.

**3 персонажа:**
- мужской род (der) → Мужчина (с отличительными чертами: усы, очки…)
- женский род (die) → Женщина
- средний род (das) → Ребёнок (мальчик/девочка ~7 лет, первоклашка)

**Цикл по теме (напр. «Быт»):**
1. Слова темы делятся по роду → 3 группы (der / die / das).
2. На каждую группу — история, где участвуют только предметы этого рода вокруг своего персонажа (мужчина сидит за столом, перед ним кофе…). Можно добавлять «гостевые» предметы из смежных тем для связности (помечать).
3. История = промпт для генерации картинки (ChatGPT). История и картинка — об одном и том же.
4. Под картинкой — короткий текст А1 с подходящими глаголами (все предметы — нужного рода), который можно **прослушать** (TTS).
5. Картинка с подписями к предметам. Кнопка «глазик» скрывает подписи → ученик называет/вставляет слова сам.
6. В конце темы — большая сводная картинка со всеми тремя персонажами и предметами вразброс; ученик называет род и слово каждого. Текст под ней — со всеми родами вперемешку.

**Глаголы:**
- Отдельный раздел + общий словарь.
- Модальные — отдельно.
- Обычные — делятся на «простое прошедшее» и «сложное прошедшее», внутри — по группам.
- К каждому глаголу: формы по лицам + прошедшее время. Текст с разными формами, можно прослушать.
- Карточки для запоминания: выбрал группу (напр. 10 глаголов) → картинка-визуализация по сценарию из этих глаголов (та же схема: сценарий → ChatGPT → картинка → скрытие → текст → озвучка).

**Словари (накопительные):**
- Словарь существительных.
- Словарь глаголов: клик по глаголу → плашка с изменением по местоимениям и прошедшей формой.

## Ключевые архитектурные решения (предложены ассистентом)
1. **Подписи — отдельным слоем поверх картинки, не впечатывать в картинку.** Иначе невозможны скрытие/клик/озвучка; правки без перегенерации; генераторы врут текст на немецком.
2. **Единый источник данных + теги, без дублирования.** Слово/глагол хранится один раз; принадлежность к темам/группам — теги. Разные «витрины» (по теме / общий словарь) из одних данных.
3. **Контент предгенерируется в конечную библиотеку**, не вживую на каждого ученика. Для А1 это десятки картинок и озвучек — дёшево и предсказуемо. Аудио — предгенерированные TTS-файлы.
4. **Каждое слово — один родной раздел**, плюс может быть «гостем» в чужой истории (с пометкой).

## Модель данных (черновик)
Один источник правды + теги → разные витрины.

**Существительное:** id · слово (Stuhl) · артикль/род (der) · мн.ч. (Stühle) · перевод (стул) · родная_тема (Быт) · гостевые_темы[] · источник (Momente/Goethe) · аудио_файл · картинка_id · позиция_подписи (x,y).

**Глагол:** id · инфинитив (stehen) · перевод · тип (модальный / прошедшее простое / прошедшее сложное) · спряжение (ich/du/er…) · прошедшая_форма (Perfekt: hat gestanden + Präteritum) · темы[] · группа · аудио_файл.

**Картинка-сцена:** id · тема · род(ы) · сценарий-промпт · файл · список_подписей[] · текст_под_картинкой · аудио_текста.

## Конвейер производства темы (на примере «Быт»)
1. Собрать слова темы из Momente A1, проставить род/мн.ч./перевод, дополнить Goethe Wortliste. Источник у каждого слова. Род — проверять (критично).
2. Разбить по роду → der / die / das.
3. На каждый род: я пишу сценарий-историю (на русском на утверждение) → промпт для ChatGPT → ты генеришь картинку.
4. Разметить подписи слоем (координаты предметов) + текст А1 под картинкой.
5. Озвучить текст (TTS) → аудиофайл.
6. Сводная сцена: все 3 персонажа + предметы вразброс.
7. Собрать в интерфейс: скрытие подписей «глазиком», клик-проверка, прослушивание.

## Дизайн-решения (раунд 2, 2026-06-03)

### Постоянные персонажи (фиксируем, чтобы держались во всех сценах)
**Принцип:** ВСЁ на персонаже (одежда, аксессуары) — строго его рода. Образ = бонусный канал для слов темы «Одежда» (Kleidung, L22). Внешность одинакова из картинки в картинку → отдельная «карточка внешности» вставляется в каждый промпт.

- **der → Otto:** усатый шотландец. der Hut (шляпа), der Mantel (пальто), der Schal (шарф), der Rock (килт/юбка), der Schuh (ботинки), der Schnurrbart (усы). Любит порядок и кофе. **Без очков и рубашки** (die Brille ж, das Hemd ср).
- **die → Грета (Greta):** рыжие волосы. die Brille (очки), die Jacke (куртка), die Hose (брюки), die Mütze (шапка), die Bluse (блузка), die Tasche (сумка). Любит готовить.
- **das → дети (Kinder):** мальчик **Тео (Theo)** ~7 лет и девочка **Lina (Лина)** ~6 — оба в зелёном (das = зелёный). Их мир — игра, дети и детские вещи среднего рода. Мнемоника: «ребёнок» = das Kind, «девочка» = das Mädchen → их слова только das. Тео — почемучка. _(Имя Lina утверждено вместо «Миа», чтобы не путать с дочерью Греты Mia.)_
- _Имена/черты — предложение, можно переименовать (Тео утверждён). Дальше одни и те же._

**Зачем имена:** живой текст («Dieter sitzt…»), мнемоника на конкретике, узнавание героя из темы в тему. Не технический артефакт.

**Одна сцена vs две (пересмотрено, раунд 3):** система — это справочник + память-дворец, а не зубрёжка за присест (см. [05-vision-roadmap.md](05-vision-roadmap.md)). Поэтому делаем БОГАТУЮ цельную сюжетную сцену на персонажа, weaving максимум слов его рода; число кадров (1 или 2) — вопрос вёрстки одной непрерывной истории, не «учебной порции». Довод «7±2 → дробить» снят.

**Сквозной сюжет:** сцены связаны в сериал про героя (story bible), у Отто — любимые вещи/привычки из слов его рода. Подробно — в [05-vision-roadmap.md](05-vision-roadmap.md).

### Подраздел «Множественное число» (внутри Существительных)
Учим отдельно, после рода. Группировка **по типу окончания Pl.**: «+n/-en» · «+e/-¨e» · «+er/-¨er» · «+s» · «без изменения / умлаут». Под каждый столбик — картинка + текст (стишок/песенка) + озвучка + «глазик». Та же конвейерная схема, другой признак группировки.

### Механика «глазик» (общий компонент, везде)
Один переиспользуемый компонент, настройка «какие слова прячутся»:
- глазик у всего текста → скрыть все целевые слова сразу;
- глазик у отдельного слова → скрыть точечно;
- «целевые»: в сцене рода — подписи-существительные; в Pl. — формы мн.ч.; в обычном тексте — любые помеченные слова.
- По умолчанию глазик доступен в любом тексте (самотренировка: вспомни → открой → проверь).

### Правило сцены и рода помещения
В сцене рода подписываем и прячем **только предметы этого рода**. Помещение — фон, в «глазик» не включаем, **кроме** случая, когда помещение того же рода (тогда подпись усиливает).
Следствие: комнаты сами распределяются по персонажам по роду — die Küche/die Wohnung → Грета; das Zimmer/das Bad/das Schlafzimmer → Тео; der Flur/der Balkon/der Garten/der Keller → Otto. das Büro / die Wohnung Мужчине не подписываем.

### Сцена группы der (Мужчина = Otto) — ОДНА сцена, ОДИН текст (исправлено)
Одна богатая сцена «комната Отто» со ВСЕМИ мужскими предметами + его сигнатурные вещи (der Kaffee, der Schlüssel). Один сюжетный текст под ней (кусок сквозной истории).
Предметы: der Tisch, der Stuhl, der Sessel, der Schrank, der Teppich, der Spiegel, der Computer, der Laptop, der Bildschirm, der Stift, der Kalender, der Drucker; фон — der Garten (в окне), der Flur (дверной проём). Подписи — слоем, без текста в самой картинке.
