# 11. Как собирать ТЕМУ целиком (рецепт + что нравится пользователю)

> Цель: по этому файлу собрать новую тему **самостоятельно и полностью**, как мы сделали Темы 1–2. Читать перед стартом темы. Это то, что пользователь от меня ждёт.

## Что значит «тема готова» (чек-лист результата)
Пользователь открывает тему — и там есть ВСЁ, «как мы любим»:
- [ ] **Хаб темы** `thema-0N.html` — выбор рода (der/die/das) с фото героя и словами; навигация между темами (предыдущая слева, будущие «скоро»); внизу — глаголы, «другое», тексты, словарь.
- [ ] **3 страницы героев** `thema-0N-der/die/das.html` (Otto / Грета / Тео-Lina): картинка с серыми плашками-словами, слова сцены (озвучка + скрыть перевод), текст (DE + RU по кнопке + озвучка), вопросы, **аудирование**, упражнения (карточки, соотнеси, напиши с артиклем, вставь в текст).
- [ ] **Глаголы темы** `verben-thema-0N.html` — список ✅/⚠️ с формами и Perfekt (справка), текст с акцентом на глаголы темы, упражнения (карточки, соотнеси, напиши перевод).
- [ ] **«Другое»** `andere-thema-0N.html?cat=...` — тренировка по категориям (цвета, оценка/свойства, …): слова с озвучкой и скрытием перевода, текст, карточки, соотнеси, напиши по-немецки.
- [ ] **Тексты темы (диалоги)** `dialog{N или 2…}.html?n=1..3` — 3 смешанных текста (все рода), картинка сверху, серые точки-маркеры на предметах, упражнения.
- [ ] **Словарь** темы наполнен в `js/woerter.js` (тема `0N`) → `dictionary.html?thema=0N` работает, плашка с кнопкой «Скачать PDF».
- [ ] **Всё озвучено живым голосом** (не робот): каждое слово/текст/реплика. Проверить, что нет fallback на браузер.
- [ ] **Навигация** между темами и героями расставлена; в шапке (chrome.js) тема включена.

## Жёсткие правила контента (НЕ нарушать)
1. **Уровень — только пройденное.** A1 (L1–L5) = **Präsens**. **Без прошедшего времени** (Perfekt/Präteritum, «gestern», «war») в тексте. Perfekt — только справкой в таблице глаголов. (см. [[level-only-praesens]])
2. **Разные лица.** Текст разнообразный: ich/du/er/sie/wir/ihr/sie/Sie, не только «я».
3. **Род — строго свой.** Когда учим слова рода (der у Otto, die у Греты, das у Тео), в тексте героя — только его род (гостевые слова другого рода помечать).
4. **Род проверять.** Артикль — ядро; ошибка рода критична. Сверять каждое слово.
5. **Сначала русский на утверждение.** Любой новый текст-история — сперва по-русски пользователю; немецкий — после «ок».
6. **Тексты разнообразные, без повторов** между собой и без воды.
7. **Фишка-слово героя:** Otto → der Preis, Грета → die Farbe, Тео → das Material (в тексте «das ist mein Wort»). Для новой темы — подобрать по смыслу.

## Правила упражнений
- **В разных текстах темы — РАЗНЫЕ слова.** Словарь темы делить на ~3 непересекающиеся группы; текст 1/2/3 тренирует свою. Касается match/sort/write/verbs. В «распредели по родам» в каждой группе — все три рода. Глаголов мало (≈7) → распределять с наименьшим повтором. (см. [[exercises-disjoint-per-text]])
- **«Показать ответ» — только после «Проверить».** Везде.
- **Аудирование:** свой голос у каждого героя; вопрос → впиши → проверь → потом ответ; кнопка «↻ Заново»; пауза/продолжение только своей кнопкой.
- **Слова сцены = слова темы** (набор над текстом сцены = чипы рода в хабе).

## Правила UI/вёрстки
- **Цвет = род:** der синий (#2563EB), die красный (#DC2626), das зелёный (#16A34A). Кнопки-переходы к героям, подписи, плашки, подсветка слов, кнопка «Словарь PDF» — в цвет рода. Фиолетовый (#6D28D9) — «другое»/служебное; жёлтый — глаголы (✅ прав./⚠️ непр.).
- **Подсветка изучаемых слов** в тексте сцены — по роду, слово с его артиклем (mein/ein/der), без надстрочной пометки.
- **Точки-маркеры на картинках диалогов** — серые (род не выдают); при раскрытии слово показывается **с артиклем** и в цвете рода. Позиции — по предметам на фото.
- **Картинки в текстах** — `<img width:100%, height:auto>` (соотношение сторон НЕ менять, не обрезать). Картинки даёт пользователь; если нужна — назвать ей **промпт** (книжная иллюстрация, без текста/букв/цифр).
- **Плашка словаря** — фиолетовая, ВНУТРИ две кнопки: «Открыть словарь» + «Скачать PDF» (`dictionary.html?thema=0N&pdf=1`).
- **Технические надписи** (промпты, «структура темы…», «данные в js/…») пользователю НЕ показывать. «Информация для разработчика» — в сворачиваемом `<details>`.
- После любой правки UI — проверить, что ничего не перекрыто/не обрезано, помещается; **и десктоп, и телефон**.
- **🔑 Согласованность:** меняешь принцип в одном месте — распространяй на ВСЕ страницы.

## Технический процесс
- **Озвучка:** `tools/gen_audio.py` (edge-tts, бесплатно). Голоса: Otto `male` (Killian), Грета `female` (Katja), дети `child` (Seraphina), продавец `seller` (Amala), девочка-Lina `childgirl`, Carla `carla`. Слова — голосом рода; диалоги — по говорящему. Добавлять новые слова темы в woerter.js (существительные берутся оттуда) и/или в списки VERBS/EXTRA/COLORS/PROPS/ANDERE и сцены SCENES; запускать `python3 tools/gen_audio.py` (готовое пропускает). Слова сцен, которых НЕТ в словаре, добавлять в EXTRA, иначе робот.
- **Анти-кэш:** css/js c `?v=N`; во всех html `<meta http-equiv="Cache-Control" content="no-cache">`; изменённую картинку переименовывать (имя+буква).
- **Проверять ДО отчёта:** `curl` живого сайта + `grep`, что новое реально отдаётся.
- **Деплой (из корня репозитория):**

## Порядок сборки темы (шаги)
1. Определить тему по `docs/09-szenen-inhalt.md` (что включать) и уровень.
2. Собрать словарь темы (der/die/das + глаголы + другое) → **сверить рода** → внести в `js/woerter.js` (тема `0N`).
3. Написать **по-русски**: 3 истории героев + 3 текста-диалога → утвердить у пользователя → перевести на немецкий (Präsens, разные лица, свой род).
4. Сгенерировать озвучку (слова, тексты, реплики, аудирования).
5. Собрать страницы по шаблонам Темы 2 (копировать и менять контент): хаб, 3 героя, глаголы, другое, диалоги.
6. Развести упражнения на 3 непересекающиеся группы.
7. Картинки — запросить у пользователя (дать промпты), вставить с сохранением соотношения сторон.
8. Включить тему в навигацию (chrome.js, главная, соседние темы).
9. Проверить на живом сайте (десктоп+телефон), отчитаться «готово, проверь» с тем, что смотреть.

## Что пользователь делает сама (её вклад)
- Утверждает русские тексты; присылает картинки (я даю промпты); даёт точечные правки по позициям плашек/точкам, голосам, сюжету. Технические решения (код, файлы, структура) — на мне, без вопросов.
