# 02. Правила проекта

## Общие (из инструкции)
1. Любое число пересчитывать; итоги в таблицах сходятся.
2. Факты — со ссылкой. Неподтверждённое помечать.
3. Не выдумывать названия/ссылки/контакты/цифры.
4. Неуверенность помечать явно.
5. Видишь ошибку/слабость → предлагай решение(я), аналитично; требовательно, но без голого критиканства.
6. Спорное — оспаривать аргументами.
7. Перед правкой — сверяться с дневником ошибок; после — дописывать.
8. Юр-вопросы (право, реклама, налоги, данные) — проверять закон + дисклеймер.

## Язык
- Общение — по-русски.
- Любой материал/копирайт — сначала по-русски на утверждение, перевод на немецкий только после «ок».
- Не путать пояснение и сам текст: на утверждение даём сам текст по-русски.
- Тексты конкретные, без воды, готовые к вставке (copy-paste).

## Порядок работы по задаче
- После постановки задачи — работаю до результата сам, без подтверждений на каждый шаг. Бытовые решения внутри задачи (как сделать, какие файлы, мелкие правки) — сам.
- Закончил — пишу «готово, проверь» и коротко что посмотреть.
- Вопрос — только когда реально нужен: не уверен (грамматика/факт/риск сломать) — спрашиваю заранее; необратимое/внешнее — всегда; новый контент — единичная сверка русского текста. Постоянных подтверждений по ходу нет.
- Подробно — в `CLAUDE.md`, раздел «Порядок работы по задаче».

## Специфика проекта (немецкий)
1. **Род существительного (der/die/das) проверять** — это ядро продукта, ошибка рода критична (🔴).
2. **Формы глаголов и прошедшее время проверять** перед публикацией.
3. История и картинка по одной группе — строго **один род** (гостевые слова из других тем помечать).
4. Уровень текстов и лексики — соответствует заявленному (А1), без выхода за уровень без пометки.
5. Любое слово в материал — только из утверждённого списка/учебника, не из головы.
