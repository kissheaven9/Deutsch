# 07. Структура сайта и интерфейс (техдокументация)

> Это то, что реализует прототип в [/site](../site). Статус: каркас v1.

## Навигация (карта сайта) — ТЕМА В ЦЕНТРЕ (зафиксировано со слов пользователя)
```
ГЛАВНАЯ = СПИСОК ТЕМ
└── выбираю тему → ТЕМА N (напр. «Знакомство и семья» = L1+L2+L3)
    │   [вверху темы — переключение разделов: Существительные ↔ Глаголы ↔ Другое ↔ Топики]
    ├── СУЩЕСТВИТЕЛЬНЫЕ — по роду (колонки der / die / das)
    │     └── выбрал колонку рода → СЦЕНА этого рода:
    │           • картинка с предметами + подписи слоем (глазик скрывает/показывает подписи)
    │           • текст-история (глазик скрывает/показывает целевые слова — вставить самому)
    │           • аудио
    │           [вверху сцены — переключение рода: der ↔ die ↔ das (та же тема)]
    │     └── Множественное число (подраздел): столбики по окончанию +(e)n | +e/¨e | +er/¨er | +s | -/¨ → картинка+стих+аудио+глазик
    ├── ГЛАГОЛЫ (по теме) — список строками: глагол · перевод · прошедшее; кнопка «форма» → спряжение по местоимениям; деление правильные/неправильные; глазик скрывает переводы; столбцы настоящее/прошедшее
    ├── ДРУГОЕ (по теме) — прилагательные/состояния, фразы, наречия/служебные, да-нет-отрицание
    └── ТОПИКИ (по теме) — ~5 накопительных текстов (все слова с темы 1 по текущую + глаголы), глазик + аудио

Глобально (доступ из меню): Словари (накопительные) · Экспорт PDF [v5]
```

> **Важно про «техническую» страницу.** Страница из трёх столбцов (слова · история · промпт ChatGPT) — это **рабочая страница ДЛЯ ПОЛЬЗОВАТЕЛЯ**: согласовать историю, проверить, что все слова взяты, и взять промпт для генерации картинки. **На самом сайте её нет** — там сцена выглядит как описано выше (картинка + подписи-слой + текст с глазиком + аудио).

## Тематики и сцены — единица запоминания (ключевой принцип)
Слова группируем НЕ по урокам учебника 1:1. Слова из разных уроков собираем в **тематику** (смысловую: «Дом», «Еда», «Город»…). Поэтому тематик меньше, чем 24 урока.
- Одна **тематика** может содержать **несколько сцен**. Пример: «Дом Otto» = сцена «Кабинет» (мебель/техника, L4+L6) + сцена «Кухня» (приборы, L16). Слово «дожидается» своей тематики, даже если его урок позже.
- **Сцена** = одна целостная тематическая картинка + кусок сквозного рассказа + слова одного рода. Это и есть единица запоминания.
- Порядок — примерно хронологический (как в учебнике), но подчинён смыслу тематики, не номеру урока.
- **Возврат из словаря:** ученик кликает слово в словаре → вспоминает сцену и персонажа («это же Otto, его любимый/он этим пользовался») → вспоминает род. Ради этого вся группировка.
- Раскладка тематик/сцен Otto на весь А1: [data/story/otto-arc-A1.md](../data/story/otto-arc-A1.md).

## Страница ТЕМЫ (вход в тему) — структура
Захожу в тему → сразу вижу всё по теме в одном месте (прототип: [/site/thema-01.html](../site/thema-01.html)):
- **Существительные — по роду:** три карточки der (Otto) / die (Грета) / das (Тео); клик по роду → его СЦЕНА (история+картинка+слова+аудио).
- **Глаголы:** правильные / неправильные (по теме; полные формы — в разделе «Глаголы»).
- **Другое — по теме:** прилагательные/состояния, фразы, наречия/служебные, да/нет/отрицание (verheiratet, zusammen, von Beruf, ein bisschen …).
Лексика темы — из Lernwortschatz (Arbeitsbuch), полностью; источник: data/themen/NN-…md.

## Страница СЦЕНЫ (ядро)
**Три столбца:** ① слова сцены (существительные · ниже глаголы · ниже другое — всё по теме) · ② текст-история (сначала весь немецкий, потом весь русский) · ③ промпт для ChatGPT.
**Картинка:** фотореалистичная сцена (как примеры пользователя Frame 6/9): персонаж + предметы одного рода, на фоне крупная надпись рода («DER»). Подписи к предметам («Der Vater» …) — отдельным слоем поверх (корректная орфография + «глазик»), в самом изображении текста не делаем.
1. **Картинка** (в v1 — плейсхолдер с предметами-эмодзи; позже — изображение из ChatGPT).
2. **Подписи поверх** — отдельный слой: артикль+слово у каждого предмета.
3. **Колонка слов** рода (с переводом).
4. **Текст-история** (А1). В v1 показываем русский черновик с пометкой «немецкий после утверждения».
5. **Аудио** — кнопка 🔊 (в прототипе — голос браузера de-DE; позже — предгенерированные TTS-файлы).
6. **Глазик** — см. ниже.

## Раздел «Глаголы» (детали — со слов пользователя, ср. [01-overview.md](01-overview.md))
- Внутри темы глаголы делятся на **правильные / неправильные**.
- Список — **строками**, не «скопом»: `глагол · перевод · прошедшее время`. Перевод нужен → строкой.
- **Глазик** скрывает переводы (учить самому → открыть → проверить).
- У каждого глагола — кнопка **«форма»**: по клику всплывает табличка **спряжения по местоимениям** (ich/du/er…).
- Два столбца времени: **настоящее** и **прошедшее**; склонение по местоимениям открывается отдельно.
- Уровень синхронен теме (без А2/будущих уроков). Полные формы/Perfekt — по мере прохождения.

## Подраздел «Множественное число» (детали)
Внутри Существительных. Группировка по типу окончания Pl.: +(e)n · +e/¨e · +er/¨er · +s · без изм./умлаут. Под каждый столбик — картинка + стих/песенка + аудио + глазик. (См. [05-vision-roadmap.md](05-vision-roadmap.md).)

## Стиль картинок
Тёплая **иллюстрация** (не фотореализм) — образец: сгенерированная картинка Otto (ED86C1A3…PNG в корне проекта). Предметы одного рода + персонаж; подписи — отдельным слоем (глазик), в самом изображении текста не делаем.

## Значки и механика «глазик» 👁
- **Глазик у отдельного слова** — скрыть/показать это слово. Скрытое = `▢▢▢`; тап по нему — открыть (самопроверка).
- **Глазик «скрыть все»** — массово скрыть все целевые слова сцены; повтор — показать все.
- **Что «целевое»:** в сцене рода — существительные-подписи; в подразделе Pl. — формы мн.ч.; в обычном тексте/топике — любые помеченные слова.
- Доступен в любом тексте по умолчанию.

## Аудио 🔊
- На слове и на тексте.
- v1: Web Speech API браузера, язык de-DE (бесплатно, работает на Mac/iPhone; набор голосов зависит от устройства).
- Позже: предгенерированные mp3 (облачный TTS) — единый голос, не зависит от устройства.

## Экспорт PDF (что можно скачать) [v5]
- словарь по теме · выбор тем → сборка · полный печатный (колонки der/die/das + глаголы отдельно) · кумулятивный · только глаголы (алфавит) · отдельный топик · блок «слова+картинка+текст» (офлайн).

## Технология прототипа
Статический сайт (HTML/CSS/vanilla JS), без сборки — открывается офлайн, легко раздать. Локальный сервер: `python3 -m http.server` с привязкой к 0.0.0.0 → доступ с телефона в той же Wi-Fi.
