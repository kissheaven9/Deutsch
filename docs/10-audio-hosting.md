# 10. Звук и публикация (решение + план)

## Проблема
Браузерный голос (Web Speech API) генерится на устройстве зрителя → у педагога по ссылке будет ЕГО голос, часто роботный. Для показа наружу не годится.

## Решение (утверждено 2026-06-04)
1. **Голос — OpenAI TTS.** Заранее «запекаем» все слова и тексты в mp3-файлы (нейроголос, естественный), кладём на сайт. Все слышат один и тот же хороший голос. Стоит копейки на наш объём.
2. **Хостинг — Cloudflare Pages / Netlify** (бесплатно). Статический сайт → загрузка папки `site/` → публичная ссылка педагогу.

## Когда делаем
Не сейчас. На этапе **MVP-показа педагогу**: Otto сцены **1–3** + сцена **Тео** (дети). До этого — локальный сервер + временный браузерный голос (быстро итерируем бесплатно).

## Что понадобится от пользователя (на этапе MVP)
- Ключ доступа **OpenAI API** (проведу по шагам; пополнение на пару $ хватит надолго).
- Возможно, аккаунт Cloudflare/Netlify (бесплатный; настрою сам, ссылку отдам).

## Как реализуем (для себя)
- Скрипт берёт списки слов/тексты сцен → OpenAI TTS (немецкий голос) → mp3 в `site/audio/<id>.mp3`.
- На страницах заменяем `speak()`/`toggleSpeak()` на проигрывание mp3 (`<audio>`), браузерный голос оставляем как запасной для локальной разработки.
- Деплой: `site/` → Cloudflare Pages (drag-and-drop или из git).
