> Живой кит (открыть в браузере): [design-kit.html](design-kit.html).

## Позиционирование
**Качественный, дружелюбный, но взрослый сервис. «Умный минимализм».**
Тёплый и визуальный (это требует метод — персонажи, истории, картинки), но оболочка вокруг контента — чистая, сдержанная, не перегруженная. Не «детский сад», не яркая аркада. Ориентир по ощущению — спокойный качественный edtech.

## Аудитория (зафиксировано)
**Первичная: подростки и взрослые 14+, уровень А1**, кто учит немецкий для жизни/учёбы и кому помогает визуально-сюжетная мнемоника.
- Обоснование: контент берётся из **Momente A1 (Hueber)** — учебника для взрослых; метод визуальный, но не инфантильный.
- Школьники 12+ попадают естественно, отдельной адаптации под них не делаем.
- **Дети 6–10 лет — не первичная цель.** Отдельный «детский режим» — потом, если понадобится (одно ядро данных, разные обёртки). Сейчас не закладываем — для MVP преждевременно.

## Кодирование рода — несущая конструкция продукта
Род (der/die/das) — ядро (🔴-критичность). Поэтому он кодируется **тремя каналами одновременно**, и **цвет — лишь один из них**:

1. **Артикль словом** — der / die / das (всегда виден).
2. **Иконка-персонаж** — мужчина / женщина / ребёнок.
3. **Цвет** — усиление, не единственный носитель.

**Зачем три канала.** Школьная палитра «красный + зелёный» — худшая пара для дальтоников (~8% мужчин их путают). Если род держится только на цвете — для них ядро ломается. Иконка + артикль снимают риск и позволяют оставить привычную школьную схему (плюс: «эффект переноса» — кто учил в классе, узнаёт коды).

**Правило:** родовые цвета — **священны, используются ТОЛЬКО для рода.** UI-кнопки, ссылки, акценты — никогда не синий/красный/зелёный из этой тройки, чтобы не путать интерфейс с кодировкой.

| Род | Персонаж | Цвет (ориентир) | Токен |
|---|---|---|---|
| **der** | Отто — мужчина (усы/очки) | синий `#2563EB` | `--gender-der` |
| **die** | Грета — женщина | красный `#DC2626` | `--gender-die` |
| **das** | Тео — ребёнок (~7 лет) | зелёный `#16A34A` | `--gender-das` |

> Имена персонажей и сквозной сюжет — в [../docs/05-vision-roadmap.md](../docs/05-vision-roadmap.md) и [../docs/06-story-rules.md](../docs/06-story-rules.md). Реализация экранов — [../docs/07-site-structure.md](../docs/07-site-structure.md) и `/site`.

Для фонов-плашек — те же цвета в светлом разведении (10–15% насыщенности), текст/иконка — насыщенным тоном.

## Цветовые токены

### Нейтрали (основа интерфейса)
| Назначение | hex | Токен |
|---|---|---|
| Фон страницы | `#F8FAFC` | `--bg` |
| Поверхность (карточки) | `#FFFFFF` | `--surface` |
| Текст основной | `#1F2937` | `--text` (не чистый чёрный — мягче, «дороже») |
| Текст вторичный | `#6B7280` | `--text-muted` |
| Границы/разделители | `#E5E7EB` | `--border` |

### Акцент UI (интерактив)
Один акцент, **вне тройки рода** — фиолетовый. Кнопки, ссылки, фокус, активные состояния.
| Назначение | hex | Токен |
|---|---|---|
| Акцент | `#6D28D9` | `--accent` |
| Акцент (hover) | `#5B21B6` | `--accent-hover` |
| Кольцо фокуса | `#6D28D9` @ 40% | `--focus-ring` |

### Семантика
| Назначение | hex |
|---|---|
| Успех (верно) | `#16A34A` ⚠️ совпадает с das — в контексте проверки рода использовать иконку ✓, не только цвет |
| Ошибка (неверно) | `#DC2626` ⚠️ совпадает с die — аналогично, иконка ✗ |

> ❓ Из-за совпадения «успех=зелёный / das» и «ошибка=красный / die» статусы проверки **обязательно** дублировать иконкой и текстом, иначе на экране проверки рода будет путаница. На прототипе проверить, не взять ли для статусов нейтральные индикаторы.

### Доступность
- Контраст текста к фону — **минимум 4.5:1** (`#1F2937` на `#FFFFFF` ≈ 12:1 — с запасом).
- Род не полагается на цвет в одиночку (см. три канала).
- Цель размера кликабельных целей — **≥ 44×44 px** (палец/мышь).

## Типографика
**Один шрифт на всё — Inter** (бесплатный, Google Fonts).
- Покрывает **кириллицу** (интерфейс русский) и **латиницу с ä ö ü ß** (контент немецкий).
- Высокая читаемость в мелких размерах — важно для подписей на картинке.

Шкала (ориентир, базовый размер 16 px):
| Роль | Размер / высота строки | Начертание |
|---|---|---|
| H1 | 30 / 36 | 700 |
| H2 | 24 / 32 | 700 |
| H3 | 20 / 28 | 600 |
| Текст | 16 / 26 | 400 |
| Мелкий / подпись | 14 / 20 | 400–500 |
| Немецкое слово в словаре | 18–20 | 600 |

Длина строки текста под картинкой — **ограничить ~60–70 знаков** (читаемость).

## Сетка, отступы, форма
- **Сетка 8 px.** Отступы кратны 4/8 (4, 8, 12, 16, 24, 32, 48).
- Много «воздуха», ограниченная ширина контентной колонки (~720 px для текста).
- **Скругления:** 8–12 px (дружелюбно, но не «детские пузыри»).
- **Тени:** мягкие, едва заметные (`0 1px 3px rgba(0,0,0,.08)`) — глубину даём сдержанно.

## Компоненты (под экраны из [01-overview.md](01-overview.md))

1. **Чип рода** — иконка-персонаж + артикль (der/die/das) + цветовой фон. Базовый «кирпич», встречается везде.
2. **Карточка-сцена** — чистая картинка + **слой подписей поверх** (подписи не впечатаны — см. архитектурное решение №1 в overview).
3. **Подпись-плашка** на картинке — слово + артикль, цвет по роду; кнопка **«глазик»** скрывает/показывает все подписи (режим самопроверки).
4. **Кнопка озвучки** — иконка-динамик у текста и у слова (TTS-файл).
5. **Плашка слова в словаре** — клик по существительному/глаголу раскрывает: род (чип), мн.ч., перевод; для глагола — спряжение по лицам + прошедшая форма.
6. **Сводная сцена** (конец темы) — все 3 персонажа, предметы вразброс; роды кодируются теми же тремя каналами.

## Что НЕ делаем
- Не используем родовые цвета (синий/красный/зелёный) для кнопок и навигации.
- Не полагаемся на цвет как единственный признак рода или статуса.
- Не впечатываем подписи в картинку.
- Не делаем «детский» визуал (маскоты, мультяшная типографика, кислотные цвета).
- Не перегружаем экран — один смысловой акцент за раз.

## Открытые вопросы
- ❓ Точные hex родовых цветов и акцента — финализировать на прототипе сцены «Быт».
- ❓ Индикаторы статуса «верно/неверно» — оставить зелёный/красный (с иконкой) или взять нейтральные, чтобы не конфликтовать с das/die.
- ❓ Стиль иконок-персонажей (плоские/линейные; степень детализации усов-очков-первоклашки).
- ❓ Тёмная тема — нужна ли в v1 (предлагаю: нет, потом).
